## 2024-05-24 - Optimizing PricingModel Cache Invalidation
**Learning:** Adding a short-circuit length check (`len(keys) != len(dict)`) before an O(N) set equality check (`set(keys) != set(dict.keys())`) to detect mock changes can actually degrade steady-state performance on hot paths. If the dictionary length rarely changes (like `cls.RATES` in production), the "fast path" fails, forcing the application to evaluate both the O(1) length check AND the O(N) set creation.
**Action:** Before optimizing conditional checks, consider the "hot path" frequency. If a condition is almost always false in production, adding it before an expensive operation only adds overhead. A better approach for such cache invalidation would be tracking a version integer or caching the keys directly to avoid set creation entirely in the steady state.
